🎶 Welcome to the Administration Panel

This panel allows you to manage the website’s musical content and data. The main available operations are:

  • ➕ Add records: use the Create button in the section list.
  • ✏️ Edit records: click Edit next to the item to update.
  • 🗑️ Delete records: click Delete (note: if a record is linked by relations, the model’s cascade rules will apply).

📂 Available Sections

  • 👤 Profile: main info (name, bio, image, contacts, socials, music channels). Note: each profile can have only one Contact, MusicChannels, and Socials record (1–1 relation with unique=True).
  • 💿 Album: title, description, cover image, links (Spotify, YouTube, Apple). Tip: links must belong to authorized domains.
  • 📜 Board Messages: news or updates with automatic created_at.
  • 📄 Page Description: main site text, only one record allowed.
  • 🏠 Address: full address, reusable by Places. Tip: validate CAP (numeric only, max 10 characters) and street number.
  • 📍 Place: venue name, linked address, Google Maps link. Note: one place may host multiple concerts.
  • 🎟️ Tickets: a ticket may be free (is_free) or have a unique link. Note: 1–1 relation with concert (unique=True).
  • 📅 Concert Schedule: date and times. Rule: start time must be before end time. No overlapping concerts allowed in the same place.
  • 🎤 Concerts: link Place, Ticket, Schedule, optionally a Tour. Note: you cannot assign two tickets to the same concert.
  • 🗺️ Tours: name, description, linked concerts. Rule: TourSchedule must contain all dates of associated concerts.

🛡️ Validation Rules

  • Text (name, bio, descriptions): min/max length; HTML and SQL not allowed (FormValidators.no_html, FormValidators.no_sql).
  • Email: must have valid email format.
  • Phone: digits only (7–15), optional leading “+”.
  • Links: must be http:// or https://, validated against allowed domains (Spotify, YouTube, Apple, Facebook, Instagram, Google Maps).
  • Image files: only .jpg, .png, .gif.
  • Ticket: unique link if provided. Only one ticket per concert.
  • Concert Schedule: date required, start/end times consistent.
  • Tour: start date ≤ end date.
